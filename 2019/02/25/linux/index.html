内存寻址

简介

本节介绍寻址技术。

内存地址

在编程中偶尔会使用内存地址作为访问内存单元内容的一种方式，当使用80x86处理器时，需要区分三种不同的地址:

  • 逻辑地址

    每一个逻辑地址都由一个段(segment)和偏移量(offset)组成。

  • 线性地址(虚拟地址)

    是一个32位无符号整数，可以用来表示高达4GB(2^32)的地址。

  • 物理地址

    用于内存芯片内存单元寻址。

内存控制单元通过一种称为分段单元的硬件电路把一个逻辑地址转换成线性地址，接着，再通过一种称为分页单元的硬件电路把线性地址转化成一个物理地址。

逻辑地址—(分段单元)—>线性地址—(分页单元)—>物理地址

分段

分段的原因

在早期，地址总线有20根，可以寻址1M(2^20)的地址空间，但是寄存器是16位的，仅能寻址(64K)的地址空间。为了解决16位的寄存器对20位的地址总线的寻址问题。因此将1M的地址空间划分成2^16个区域，这样只要将寄存器的数据左移4位(等于乘上16)就能构成20位的地址，访问到所有区域的首地址(因为左移4位=乘16，所以这个首地址必然是16的倍数)，然后再加上另一个寄存器中16位的偏移量就可以访问到1M地址空间中任何地址(因此一个段最大只有64KB)。

段选择符

前面说过，一个逻辑地址由段标识符和段内的偏移量组成。段标识符是一个16位的字段，称为段选择符。偏移量是一个32位的字段。

图3

其中索引号指定段描述符表中的段描述符，TL指定是从全局描述符表中找还是局部描述符表中找，RPL指定请求者的特权级别。

段寄存器

为了快速方便地找到段选择符，处理器提供了段寄存器，用来存放段选择符。有cs，ss，ds，es，fs和gs，其中:

cs 代码段寄存器，指向包含程序指令的段。

ss 栈段寄存器，指向包含当前程序栈的段。

ds 数据段寄存器，指向包含静态数据或全局数据段。

其余三个作一般用途，可以指向任意的数据段。

cs寄存器还有一个重要的功能就是它的RPL字段，用来指明CPU当前的特权级(CPL)。0表示最高级，3表示最低级，Linux只用0和3表示内核态和用户态。

段描述符

每个段都由一个8字节的段描述符表示，它描述了段的特征，段描述符又被放在全局描述符表(GDT)或者局部描述符表(LDT)中。

通常CPU只定义一个GDT，如果进程需要额外的段，可以创建自己的LDT。GDT的地址和大小存放在gdtr控制寄存器中，当前正在被使用的LDT地址和大小则存放在ldtr控制寄存器中。

图4
字段 描述
Base(共32位) 包含段的首字节的线性地址
G 粒度标志:如果该位为0，则段大小以字节为单位，否则以4096字节(4KB)的倍数计
D/B D表示段是代码段，B则表示段是数据段
AVL 可以由操作系统使用，但是Linux忽略此字段
Limit(共20位) 存放段中最后一个内存单元的偏移量，从而决定段的长度。如果G=0，则一个段的大小是1B～1MB，G=1则为4KB～4GB。
P segment-presen标志:等于0表示段当前不内存中
DPL 描述符特权级字段:用于限制对这个段的存取，它表示访问这个段要求CPL，例如DPL位0，这个段只有CPL位为0才能访问。
S 系统标志:0代表是系统段，1代表是普通的代码段或数据段
Type 描述了段的类型特征和存取权限
快速访问段描述符

根据图2所示，每次逻辑地址转换都需要根据段选择符访问内存中的段描述符，为了加快段描述符的访问，80x86处理器提供了一种附加的非编程寄存器来供6个段寄存器使用。每个非编程寄存器含有一个的段描述符(8字节/8B)。当一个段选择符被装入段寄存器时，它所指定的段描述符就装入对应的非编程寄存器。至此，针对这个段的逻辑地址转换就不需要再访问GDT或者LDT，而是直接访问存放段描述符的非编程寄存器即可。

由于一个段描述符是8B长，所以它在内存中的地址是

描述符表的首地址+段选择符索引号(前13位)
公式1

而描述符表的首地址在上文中提过，存储在gdtr控制寄存器或者ldtr控制寄存器中(取决于描述符表是GDT还是LDT)。

综合上面的所有知识，一个逻辑地址的转换可以由下图表示

  • 从段选择符的索引号和描述符表首地址计算出描述符的地址(见公式1)，

Linux中的分段

虽然80x86在硬件上鼓励分段，但是Linux对分段的使用非常有限。实际上，分段和分页在某种程度上有点多余:分段可以给每个进程分配不同的线性地址空间，而分页可以把同一线性地址空间映射到不同的物理地址。所以Linux更偏爱分页的方式。

以下是四个主要的Linux段的段描述符字段的值

段 Base G Limit S Type DPL D/B P
用户代码段 0x00000000 1 0xfffff 1 10 3 1 1
用户数据段 0x00000000 1 0xfffff 1 2 3 1 1
内核代码段 0x00000000 1 0xfffff 1 10 0 1 1
内核数据段 0x00000000 1 0xfffff 1 2 0 1 1

可以看出四个段除了Type字段和DPL字段稍有不同之外，其余的字段全部是相同的，线性地址空间都达到了4GB(2^20*4KB)，这意味着用户态和内核态下的所有进程都可以使用相同的逻辑地址。

所有段的基地址都是0x00000000，根据线性地址计算公式

Base+逻辑地址段偏移量=线性地址

可得

逻辑地址段偏移量=线性地址

即Linux下逻辑地址和线性地址是一致的。

相应的段选择符由宏__USER_CS，__USER_DS，__KERNEL_CS，__KERNEL_DS分别定义。例如，为了对内核代码段寻址时，内核只需要将__KERNEL_CS宏产生的值装进cs段寄存器即可。

Linux GDT

在单处理系统中只有一个GDT，在多处理器系统中，每个CPU都由一个GDT。所有的GDT都存放在一个cpu_gdt_table数组中，所有的GDT的地址和大小都存在cpu_gdt_descr数组中。

每个GDT包含18个段描述符和14个空的，未使用的，或者保留的项。插入未使用的项是为了凑齐32字节，使得经常使用的描述符能够放入32字节的硬件高速缓存中。

图6

每一个GDT包含的18个段描述符指向下列的段

  • 3个局部线程存储(TLS)段:这种机制允许多线程应用使用最多3个局部于线程的数据段。

  • 4个用户态和内核态的代码段和数据段。

  • 任务状态段(TSS):每个处理器有一个。每个TSS相应的线性空间地址都是内核数据段相应线性地址空间的一个小子集。所有的任务状态段都顺序地存放在init_tss数组中，第n个CPU的TSS描述符的Base字段指向init_tss[n-1]。
  • 一个缺省段局部描述符表段，这个段通常被进程共享。
  • 5个PNPBIOS code/data段。
  • 3个APMBIOS code/data段。
  • 一个内核用来处理双重错误异常的TSS段(double fault TSS)。
Linux LDT

大多数用户态下的程序都不使用局部描述符表，内核定义了一个缺省的LDT供多数进程共享。缺省的局部描述符表存放在default__ldt数组中。

分页

硬件中的分页

前面说过，分页单元把线性地址转换成物理地址，其中一个关键的任务就是把所请求的访问类型与线性地址的访问权限做比较。

为了效率，线性地址被分成固定长度的组，称为页(page)。页内部连续的线性地址被映射到连续的物理地址空间上。

分页单元把所有的RAM分成固定长度的页框(page frame)。每一个页框包含一个页。

把线性地址映射成物理地址的数据结构叫做页表(page table)，页表存放在内存中，并在启动分页单元之前必须由内核对页表进行初始化。

常规分页

32位的线性地址被分成3个域:

Directory(目录):最高10位

Table(页表):中间10位

Offset(偏移量):最低12位

因此线性地址的转换需要两步，每一步都需要基于一种转换表:

  1. 根据Directory查找页目录表获得页表物理地址
  2. 根据第一步中查找查找到的页表物理地址和Table找到页物理地址
  3. 根据页地址和偏移量获得物理地址
图7

正在使用的页目录的物理地址被存放在控制寄存器cr3中。

页目录项和页表项有相同的结构

字段 描述
Filed 包含页框物理地址的前n(最高20)位。
Global 只用于页表项，用来防止页从TLB中被刷新出去。
Page Size 只用于页目录项，如果为1，则目录项指项2MB或4MB的页框。
Dirty 只用于页表项，判断页框是否被修改过。
Accessed 当分页单元对页框进行寻址时就设置这个标志。
PCD 控制硬件高速缓存处理页表或者页的方式。
PWT 同上。
U/S 访问页表或者页的特权级。
R/W 页表或者页的存取权限(R R+W)。
Present Present=1表示所指的页在主存中，反之则不在。
拓展分页

拓展分页允许页框大小为4MB，因此需要22位Offset。

32位的线性地址被分成3个域:

Directory(目录):最高10位

Offset(偏移量):最低22位

图8

拓展分页下，没有页表项，页目录项就可以直接获得页框的物理地址。

拓展分页和常规分页的页目录项大致相同，除了:

Filed字段只有前10位有效。

Page Size设置成为1。

硬件保护方案

分页单元和分段单元的保护方案不同，页和页表相关的特权级只有两个，User/Supervisos(U/S)标志控制。此外，页的存取权限只有读和写两种，而段有读，写和执行三种。

物理地址拓展分页机制

处理器所支持的RAM容量收到地址总线上的地址管脚数目限制，前面说过，20位的地址总线就只能寻址1MB的RAM，32位理论上可以寻址4GB的RAM。显然，通过提升地址总线可以提升寻址RAM容量。

因此，早期Inter通过把管脚从32增加到36，使得寻址能力达到了64GB，但是线性地址还是32位的，因此需要引入一种新的分页机制，把32位的线性地址转换为36位物理地址，这就是物理地址拓展(PAE)机制。

通过cr4控制寄存器的PAE标志激活PAE，页目录项中的Page Size设置为1，分页机制发生改变

  • 64GB的RAM被分为2^24个页框。
  • 页表项的Filed字段从20位拓展到24位，因此页表项大小从32位变为64位，一个4KB的页表只能包含512个表项。
  • 引入一个页目录指针表(PDPT)的页表新级别，它由4个64位的表项组成。
  • cr3控制寄存器包含一个27位的页目录指针表(PDPT)基地址。

当线性地址映射到4KB的页时(Page Size=0)，32位线性地址:

  • cr3:指向一个PDPT
  • 位31-30:指向PDPT中4个项中的一个
  • 位29-21:指向页目录表512项中的一个
  • 位20-12:指向页表中512项中的一个
  • 位11-0:页中的偏移量
图9

当线性地址映射到2MB的页时(Page Size=1)，32位线性地址:

  • cr3:指向一个PDPT
  • 位31-30:指向PDPT中4个项中的一个
  • 位29-21:指向页目录表512项中的一个
  • 位20-0:页中的偏移量
图10

需要注意的是，上面提到的拓展分页和物理地址拓展的差别:

拓展分页是对于线性地址和物理地址位数相同的情况下，来划分不同页框大小使用的，目的是将大段连续的线性地址转换成相应的物理地址。

物理地址拓展是拓展了物理地址，线性地址依旧是32位，而物理地址位36位，其目的是使32位地址空间能被转换成36位物理地址从而寻址到64GB的RAM。物理地址拓展并没有增加线性空间的大小，因此用户态的进程能使用的最大物理地址空间仍旧是4GB。

64位系统中的分页

在32位系统中，普遍采用两级分页，然而在64位系统中，两级分页并不适用。

因为这样页目录项和页表项太庞大，因此64位系统普遍采用三级甚至四级分页。

x86_64平台使用48位来寻址，采用四级分页，线性地址分级为9+9+9+9+12。

硬件高速缓存

我们知道CPU的时钟频率有几个GHz，而动态RAM(DRAM)芯片的存取时间是时钟周期的数百倍，这意味着从RAM中读写数据时，CPU需要等待很长的时间。

未来弥补CPU和RAM之间的速度差距，引入了硬件高速缓存。

硬件高速缓存给予著名的局部性原理，由于程序的循环结构及数组可以组织成线性数组，最近最常用的相邻地址在不久的将来又被用到的可能性极大。因此，引入小而快的内存来存放最近最常使用的代码和数据就可以不用多次访问RAM。

图11

高速缓存单元被放置在分页单元和内存之间，它包含一个硬件高速缓存和一个高速缓存控制器。

高速缓存控制器存放一个表项数组，每个表项对应高速缓存内存中的一个行。每个表项有一个标签(tag)和描述高速缓存行状态的几个标志(flags)。这些标签由一些位组成，高速缓存控制器可以通过这些位辨别由这个行所映射的内存单元。

关于缓存的更多详情可以继续阅读计算机组成原理一书。

处理器的cr0寄存器的CD标志位用来启用或者禁用高速缓存电路，NW标志用来指明高速缓存使用通写还是回写策略。

在前面的页目录项中，有PCD和PWT两个字段。PCD(Page Cache Disable)用来指明访问这个页框中的数据时，高速缓存必须被启用还是禁用。PWD(Page Write-Through)指明页框数据被修改时，使用回写策略还是通写策略。

Linux清除来所有PCD和PWT，因此对所有页框都启用高速缓存，而且使用回写策略。

TLB

除了通用的高速缓存外，80x86处理器还包含一个TLB(Translation Lookaside Buffer)的高速缓存用于加快线性地址的转换。

当一个线性地址在第一次使用时，不在高速缓存内，因此访问RAM中的页表计算出物理地址，然后将这个物理地址加入到TLB表项中，再次访问这个线性地址可以直接TLB中获取物理地址。

当cr3控制寄存器被修改时，TLB中所有的数据都会失效，这是因为cr3存储的是页目录基地址，修改cr3说明启用新的一组页表，而TLB指向旧页表中的数据。
